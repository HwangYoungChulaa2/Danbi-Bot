# 룰렛 시스템 업데이트 가이드

## 업데이트 내용

### 1. 사용자 프로필 닉네임 표시 개선
**변경 전:**
```
🎰 익명(sum)님의 결과
= 테스트333 =
```

**변경 후:**
```
🎰 단비(sum)님의 결과
= 테스트333 =
```

- 룰렛 실행 시 실제 닉네임이 표시됩니다
- `!킵` 명령어 실행 시에도 실제 닉네임이 표시됩니다
- 고유닉(UID)을 기준으로 사용자를 식별하므로 닉네임이 변경되어도 데이터가 유지됩니다

### 2. DJ용 유저 관리 시스템 추가

#### 유저 목록 표시
- 룰렛 시스템 설정 → "유저 관리" 탭에서 룰렛을 돌린 사용자 목록 확인
- 각 사용자는 `닉네임(고유닉)` 형식으로 표시됩니다
  ```
  1. 단비(sum)
  2. 비비(dsfsd)
  3. 양이(33f4)
  ```

#### 유저별 킵 목록 관리
- 유저 목록에서 사용자 클릭 → 상세 페이지 진입
- **킵 목록 (DJ 관리)** 섹션에서 DJ가 직접 킵 항목을 관리할 수 있습니다
  ```
  단비(sum) 킵목록
  1. 양양 X5
  2. 나비
  3. 수수
  ```

#### DJ 관리 기능
- **킵 항목 추가**: 항목 이름과 개수를 입력하여 추가
- **킵 항목 수정**: 기존 항목의 이름과 개수 수정
- **킵 항목 삭제**: 불필요한 항목 삭제

### 3. 닉네임 자동 업데이트
- 고유닉(UID)을 기준으로 사용자 데이터 관리
- 사용자가 닉네임을 변경하면 다음 룰렛 실행 시 자동으로 업데이트
- 기존 룰렛 히스토리와 킵 목록은 모두 유지됩니다

## 데이터 구조

### 사용자 데이터
```javascript
{
  "sum": {  // 고유닉 (UID) - 절대 변경되지 않음
    "uid": "sum",
    "nickname": "단비",  // 실제 닉네임 - 자동 업데이트
    "history": [  // 룰렛 결과 히스토리
      {
        "timestamp": "2026-02-19T...",
        "rouletteName": "테스트룰렛",
        "result": "테스트333",
        "triggerMessage": "!테스트"
      }
    ],
    "customNotes": [],  // DJ가 추가한 메모
    "keepList": [  // DJ가 관리하는 킵 목록 (새로 추가)
      {
        "id": "unique-id-1",
        "name": "양양",
        "count": 5,
        "addedBy": "DJ",
        "timestamp": "2026-02-19T..."
      },
      {
        "id": "unique-id-2",
        "name": "나비",
        "count": 1,
        "addedBy": "DJ",
        "timestamp": "2026-02-19T..."
      }
    ]
  }
}
```

## 사용 방법

### 1. 룰렛 실행 시 닉네임 확인
1. 사용자가 룰렛 트리거 메시지 입력
2. 시스템이 자동으로 실제 닉네임을 가져와서 표시
3. 결과: `🎰 단비(sum)님의 결과 = 테스트333 =`

### 2. !킵 명령어로 킵 기록 조회
1. 사용자가 `!킵` 입력
2. 시스템이 해당 사용자의 킵 기록을 실제 닉네임과 함께 표시
3. 결과:
   ```
   📋 단비(sum) 님킵기록
   1. 양양 X5
   2. 나비
   3. 수수
   ```

### 3. DJ가 유저별 킵 목록 관리
1. 메인 창에서 "룰렛 시스템" 설정 버튼 클릭
2. "유저 관리" 탭 선택
3. 관리할 사용자 클릭 (예: 단비(sum))
4. "킵 목록 (DJ 관리)" 섹션에서:
   - **킵 항목 추가**: "킵 항목 추가" 버튼 클릭 → 이름과 개수 입력
   - **킵 항목 수정**: 항목 옆 "수정" 버튼 클릭 → 새 이름과 개수 입력
   - **킵 항목 삭제**: 항목 옆 "삭제" 버튼 클릭 → 확인

### 4. 닉네임 변경 시 자동 업데이트
1. 사용자가 스푼캐스트에서 닉네임 변경 (예: 단비 → 단비님)
2. 다음 룰렛 실행 시 자동으로 새 닉네임으로 업데이트
3. 기존 데이터는 모두 유지 (고유닉 기준으로 관리)

## 주의사항

1. **고유닉(UID)이 중요합니다**
   - 고유닉은 절대 변경되지 않으므로 사용자 식별의 기준이 됩니다
   - 닉네임은 변경될 수 있지만 고유닉은 변경되지 않습니다

2. **킵 목록과 룰렛 히스토리의 차이**
   - **룰렛 히스토리**: 실제 룰렛 실행으로 생성된 결과 기록
   - **킵 목록 (DJ 관리)**: DJ가 수동으로 관리하는 별도의 킵 목록
   - 두 목록은 독립적으로 관리됩니다

3. **데이터 백업**
   - "유저 관리" 탭에서 "전체 데이터 다운로드" 버튼으로 백업 가능
   - 정기적으로 백업하는 것을 권장합니다

   - `addKeepItem()`: 킵 항목 추가 함수

   - `deleteKeepItem()`: 킵 항목 삭제 함수

## 테스트 시나리오

### 시나리오 1: 닉네임 표시 테스트
1. 사용자가 룰렛 트리거 메시지 입력
2. 결과 메시지에 실제 닉네임이 표시되는지 확인
3. `!킵` 명령어 실행 시 실제 닉네임이 표시되는지 확인

### 시나리오 2: 킵 목록 관리 테스트
1. 룰렛 시스템 설정 → 유저 관리 탭 이동
2. 사용자 선택 (예: 단비(sum))
3. "킵 항목 추가" 버튼 클릭 → "양양", 개수 "5" 입력
4. 킵 목록에 "양양 X5" 표시 확인
5. "수정" 버튼으로 개수 변경 테스트
6. "삭제" 버튼으로 항목 삭제 테스트

### 시나리오 3: 닉네임 자동 업데이트 테스트
1. 사용자의 현재 닉네임 확인 (예: 단비)
2. 룰렛 실행 → 결과에 "단비(sum)" 표시 확인
3. 스푼캐스트에서 닉네임 변경 (예: 단비 → 단비님)
4. 다시 룰렛 실행 → 결과에 "단비님(sum)" 표시 확인
5. 기존 데이터가 유지되는지 확인

## 문제 해결

### Q: 닉네임이 "익명"으로 표시됩니다
**A**: 고유닉을 가져오지 못한 경우입니다. 스푼캐스트 페이지가 완전히 로드된 후 다시 시도하세요.

### Q: 킵 목록이 표시되지 않습니다
**A**: 기존 데이터의 경우 keepList 필드가 없을 수 있습니다. 룰렛을 한 번 실행하면 자동으로 추가됩니다.

### Q: 닉네임 변경이 반영되지 않습니다
**A**: 닉네임은 룰렛 실행 시마다 자동으로 업데이트됩니다. 룰렛을 한 번 실행해보세요.

## 업데이트 적용 방법

1. 기존 Danbi-Bot 종료
2. 수정된 파일로 교체:
   - `engine/modules/rouletteBot.js`
   - `index.html`
3. Danbi-Bot 재실행
4. 기존 데이터는 자동으로 마이그레이션됩니다
